# Prompt Engineering Strategy for ORII Calendar Fetching

## Test Query Categories (15 Prompts for Testing)

### Category 1: General/Unrelated Queries (3-4 queries)
1. "Hi, how are you today?"
2. "What calendars do you have access to?"
3. "How does this assistant work?"
4. "What's the weather like?"

### Category 2: Basic Time-Based Fetching (4-5 queries)
5. "What do I have today?"
6. "What's on my schedule tomorrow?"
7. "Show me next week's meetings"
8. "What do I have this weekend?"
9. "Any appointments next Monday?"

### Category 3: Semantic Event Matching (4-5 queries)
10. "When was my last therapy session?" (should find "Dr. Smith appointment" or "Mental health check-in")
11. "When is my next gathering with friends?" (should find "Birthday party", "Dinner with Sarah", etc.)
12. "When did I last meet with john@company.com?" (should match attendee emails)
13. "Show me my dental appointments" (should find "Dentist", "Dr. Jones DDS", "Teeth cleaning")
14. "When is my next workout?" (should find "Gym", "Yoga class", "Personal training")

### Category 4: Complex Contextual Queries (1-2 queries)
15. "What meetings do I have with my manager this week?" (needs to understand relationships)

## Core Prompt Engineering Strategy

### 1. Intent Classification Prompt

```python
INTENT_CLASSIFICATION_PROMPT = """
You are ORII, an intelligent calendar assistant. Analyze the user's query and classify their intent.

Current date and time: {current_datetime}
User timezone: {user_timezone}

User query: "{user_query}"

Classify the intent into ONE of these categories:

1. GENERAL_CHAT - Non-calendar related questions (greetings, how are you, weather, etc.)
2. CALENDAR_INFO - Questions about calendar access, how the system works
3. FETCH_EVENTS_TIME - Time-based calendar queries (today, tomorrow, next week, etc.)
4. FETCH_EVENTS_SEMANTIC - Semantic event searches (therapy, dentist, meetings with person X)
5. CLARIFICATION_NEEDED - Query is too ambiguous to process

Examples:
"Hi how are you?" → GENERAL_CHAT
"What calendars can you access?" → CALENDAR_INFO  
"What do I have tomorrow?" → FETCH_EVENTS_TIME
"When was my last therapy session?" → FETCH_EVENTS_SEMANTIC
"Show me meetings" → CLARIFICATION_NEEDED (missing time context)

Return ONLY the intent category name.
"""
```

### 2. Semantic Event Matching Prompt

```python
SEMANTIC_MATCHING_PROMPT = """
You are analyzing calendar events to find semantic matches for a user query.

User timezone: {user_timezone}
Current date: {current_date}

Here are the user's calendar events:
{calendar_events_json}

Your task:
1. Understand what TYPE of event the user is looking for semantically

3. Consider synonyms, related terms, and context

Semantic matching guidelines:
- "therapy" matches: "Dr. Smith", "Mental health", "Counseling", "Therapist visit"
- "workout" matches: "Gym", "Yoga", "Fitness", "Personal training", "Exercise class"
- "dentist" matches: "Dr. Jones DDS", "Dental cleaning", "Teeth appointment"
- "dinner" matches: "Dinner with X", "Restaurant", "Meal with", "Eating out"
- "meeting with [person]" matches: events where that person is an attendee or mentioned

Return a JSON response:
{{
  "matches": [
    {{
      "event_id": "event_123",
      "title": "Event Title",
      "start_time": "2025-06-08T15:00:00",

      "match_reason": "Event title contains 'Dr. Smith' which semantically matches 'therapy session'"
    }}
  ],
  "total_matches": 1
}}

If no semantic matches found, return empty matches array.
"""
```

## Implementation Strategy

### Function Flow Design:
```python
async def process_calendar_query(user_query: str, user_context: dict):
    # Step 1: Classify intent
    intent = await classify_intent(user_query, user_context)
    
    if intent == "GENERAL_CHAT":
        return await handle_general_chat(user_query)
    
    elif intent == "CALENDAR_INFO":
        return await handle_calendar_info(user_query)
    
    elif intent == "FETCH_EVENTS_TIME":
        time_params = await extract_time_parameters(user_query, user_context)
        if time_params["confidence"] < 0.7:
            return await request_clarification(user_query, "unclear_time")
        events = await fetch_events_by_time(time_params)
        return await generate_conversational_response(user_query, intent, events)
    
    elif intent == "FETCH_EVENTS_SEMANTIC":
        events = await fetch_all_recent_events(user_context)  # Get larger dataset
        matches = await semantic_event_matching(user_query, events)
        return await generate_conversational_response(user_query, intent, matches)
    
    elif intent == "CLARIFICATION_NEEDED":
        return await request_clarification(user_query, "ambiguous_query")
```

### Key Implementation Notes:

1. **Two-Stage Fetching for Semantic Queries**: 
   - First fetch a larger date range of events (last 6 months + next 3 months)
   - Then apply semantic matching to find relevant events

2. **Confidence Scoring**: 
   - Each prompt should return confidence scores
   - Low confidence triggers clarification requests

3. **Context Preservation**: 
   - Pass user timezone, current time to all prompts
   - Include recent query history for better context

4. **Error Handling**: 
   - Graceful degradation when LLM fails
   - Fallback to simpler keyword matching if needed

This approach leverages GPT-4's semantic understanding while providing structure and reliability for your calendar assistant.
